# 📚 Documentation Analyzer with Intelligent Crawling Agent

An **AI-powered documentation analysis system** that automatically crawls websites or ingests files, builds a **vector knowledge base**, and answers user questions using **Retrieval-Augmented Generation (RAG)**.

The system intelligently extracts knowledge from documentation, stores semantic embeddings in a **vector database**, and generates answers with **source citations** using modern LLM frameworks.

This project demonstrates **AI engineering concepts used in real-world AI products**, including intelligent crawling, document processing, semantic retrieval, and LLM-powered reasoning.

---

# 🚀 Key Features

### 🤖 Intelligent Web Crawling

Automatically crawls documentation websites and extracts useful content while filtering out irrelevant pages such as login pages, privacy policies, and marketing pages.

### 🧹 Document Processing & Chunking

Extracted content is cleaned and divided into **semantic chunks** for optimal embedding and retrieval.

### 🧠 Semantic Search

Uses **vector embeddings** to find relevant information based on meaning rather than simple keyword matching.

### 📑 Metadata-Enriched Knowledge Base

Each chunk is stored with metadata including:

* Source URL
* Document title
* Section
* Document type

This allows **precise and context-aware retrieval**.

### 💬 Conversational Question Answering

Users can ask questions in natural language and receive contextual answers generated by an LLM.

### 🔎 Source Attribution

All answers include references to the original document sources to improve trust and transparency.

### 🎨 Interactive Streamlit Interface

A clean and modern UI allows users to:

* Crawl documentation
* Upload local files
* Ask questions
* View answer sources
* Track session history

# 🏗 System Architecture

## Knowledge Ingestion Pipeline

Seed URLs / Files
→ Intelligent Web Crawler
→ HTML / PDF Parsing
→ Content Cleaning
→ Document Chunking
→ Metadata Enrichment
→ Embedding Generation
→ Vector Database (ChromaDB)

## Query Pipeline

User Question
→ Query Embedding
→ Vector Similarity Search
→ Metadata Filtering
→ Context Retrieval
→ LLM Generation
→ Answer + Source Citations

# ⚙ Installation

## 1️⃣ Clone the Repository

```
git clone [https://github.com/your-username/doc-analyzer.git](https://github.com/KartikRaut09/Documentation-Analyzer-with-Intelligent-Crawling-Agent)
cd doc-analyzer
```

---

## 2️⃣ Create Virtual Environment

```
python -m venv venv
```

Activate environment

### Windows

```
venv\Scripts\activate
```

### Mac / Linux

```
source venv/bin/activate
```

---

## 3️⃣ Install Dependencies

```
pip install -r requirements.txt
```

---

## 4️⃣ Configure Environment Variables

Create a `.env` file:

```
GROQ_API_KEY=your_groq_api_key
GROQ_MODEL=llama-3.1-8b-instant
OPENAI_API_KEY=your_openai_key_if_using_openai_embeddings
```

---

# ▶ Running the Application

Start the Streamlit application:

```
streamlit run app.py
```

Open the provided local URL in your browser.

# 🔧 Recent Improvements & Fixes

This project was updated to work with **latest library versions** and improve reliability.

### LangChain Retriever API Update

**Problem:** Newer LangChain versions removed `get_relevant_documents()`.

**Fix:** Updated retriever logic to:

```
retriever.invoke(query)
```

Fallback support for older versions was added.

---

### Environment Variable Loading Fix

**Problem:** Groq API key was not always loaded.

**Fix:** Added `load_dotenv()` in the QA module to ensure environment variables load correctly.

---

### Updated Groq Model

**Problem:** Previously used model was deprecated.

**Fix:** Model is now configurable with environment variable:

```
GROQ_MODEL=llama-3.1-8b-instant
```

---

### Streamlit API Update

**Problem:** `st.experimental_rerun()` removed in newer versions.

**Fix:** Replaced with:

```
st.rerun()
```

---

### Improved UI Error Handling

Added `try/except` blocks around:

* crawling
* indexing
* question answering

This prevents crashes and provides clear error messages.

---

### Fixed Relative Link Crawling

**Problem:** Relative links were resolved incorrectly.

Old behavior:

```
urljoin(seed_url, href)
```

New behavior:

```
urljoin(current_page_url, href)
```

This ensures proper crawling of documentation sites.

---

### Improved Crawler Reliability

Enhancements:

* Added **User-Agent header**
* Increased request timeout
* Added `response.raise_for_status()` to detect failed requests

This improves crawling stability.

---

### Vector Database Safety Check

**Problem:** Chroma crashed when no chunks were generated.

**Fix:** Added validation to prevent empty indexing.

Now the system raises a clear error if no documents were processed.
